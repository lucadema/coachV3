# Aether Glimpse Pilot Dashboard

Read-only React dashboard for pilot-programme aggregate Glimpse telemetry.

## Local Run

Start the admin backend, which serves the public dashboard API:

```bash
./run_admin_backend.sh
```

Then start the dashboard frontend:

```bash
cd dashboard
npm install
npm run dev
```

The local dashboard runs at:

```text
http://localhost:5175
```

## URL Format

Dashboard links are generated by the admin control panel using the dashboard
token template. The frontend reads the token from either query parameter:

```text
http://localhost:5175/?t=<dashboard-token>
http://localhost:5175/?token=<dashboard-token>
```

The frontend calls:

```text
GET /dashboard/{token}
```

on the admin backend. Configure the deployed dashboard with:

```text
VITE_DASHBOARD_API_BASE_URL=https://<admin-backend-host>
```

For compatibility with the Glimpse frontend deployment config,
`VITE_API_BASE_URL` is also accepted as a fallback. The API returns aggregate
data only and never returns raw conversation text, session trails, or
individual feedback text.

## Test Mode

Use deterministic visual test data without a backend token:

```text
http://localhost:5175/?test
http://localhost:5175/?test=true
```

Test mode is generated in:

```text
src/testData/dashboardTestData.ts
```

## Debug Mode

Use debug mode when the real dashboard URL shows the generic unavailable
state:

```text
https://<dashboard-host>/?t=<dashboard-token>&debug=true
```

The debug panel shows the resolved API base URL, a redacted dashboard token,
the redacted request URL, and any fetch/HTTP status. It does not show the full
token.

## Sections

The dashboard intentionally has exactly three main sections:

- `src/components/ProblemCategorySection.tsx`
- `src/components/EngagementHealthSection.tsx`
- `src/components/ValueUnlockedSection.tsx`

Shared chart pieces live in:

- `src/components/HorizontalBarChart.tsx`
- `src/components/StackedSignalBar.tsx`
- `src/components/MetricCard.tsx`

Frontend transformation helpers live in:

- `src/data/dashboardTransforms.ts`
- `src/data/valueCalculations.ts`

The backend feedback value input calculation lives in:

```text
../admin_backend/dashboard.py
```

That helper also aggregates the `flag_to_organisation` feedback answer into
yes/no counts for the Value Unlocked subsection.

## Cost Per Hour

The hourly rate is dashboard-side only. It is not stored in the backend.

The value is persisted in local storage using a key scoped to the dashboard
token:

```text
glimpse_dashboard_hourly_rate_<token>
```
